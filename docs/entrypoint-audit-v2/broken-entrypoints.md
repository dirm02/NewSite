## Notes / non-issues (verified NOT broken)

- `providersList` (`/services/providers/provider-list`) — different **key** from
  `provider` but **same path**, which IS registered. Homepage "Popular Providers
  → View All" works. (Earlier key-based scan false-positived this; path-based
  scan clears it.)
- The live public header (`SiteHeaderRightActions` **variant 1**) uses
  `login` + `chooseSignUp`, both registered → Sign In / Join Us work. The
  `userSignup`/`register` 404s only affect alt-template header variants (2–13),
  not the live site.
- Logout (customer + provider) clears the PB session and routes to `login`
  (verified by the `@navigation` E2E suite).
